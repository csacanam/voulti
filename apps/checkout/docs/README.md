# Project Documentation Hub

The `docs/` directory now separates guidance for AI operators and human developers:

- `ai/` — Playbooks for autonomous agents modifying the codebase. Focus on guardrails, centralized configuration, and critical flows.
- `dev/` — Onboarding notes and conventions for human engineers collaborating on the project.

When contributing:
1. Decide who the change targets (AI, human devs, or both).
2. Update or create the corresponding document in `ai/` or `dev/`.
3. Keep references in sync between both sections when the knowledge impacts all contributors.
